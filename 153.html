Solution · Compare Mid vs Right (Clean 2-branch)
1
Init: left = 0, right = nums.length - 1. No early exits needed — the loop handles all cases including no rotation.
2
Only one comparison per iteration — nums[mid] vs nums[right]:
min in right half
nums[mid] > nums[right]
→ left = mid + 1
mid can't be min
min is mid or left
nums[mid] <= nums[right]
→ right = mid
mid could be the min
leftrightmidaction
0(4)6(2)3(7)7>2 → left=4
4(0)6(2)5(1)1≤2 → right=5
4(0)5(1)4(0)0≤1 → right=4
4(0)4(0)—left==right → return 0 ✓
Why right = mid (not mid−1)? When nums[mid] <= nums[right], mid itself could be the minimum. Using right = mid keeps it in the search window. Loop terminates when left == right — that's the answer.
