Solution · Binary Search for Drop Point
1
Early exits: single element → return it. If nums[right] > nums[0] the array is not rotated → return nums[0].
2
Binary search for the drop point — the unique index where nums[mid] > nums[mid+1] or nums[mid-1] > nums[mid]:
Drop after mid
nums[mid] > nums[mid+1]
→ return nums[mid+1]
Drop before mid
nums[mid-1] > nums[mid]
→ return nums[mid]
Mid in left half
nums[mid] > nums[0]
→ left = mid+1
Mid in right half
otherwise
→ right = mid-1
Why track nums[0]? In a rotated array, the left portion is always greater than nums[0] and the right portion is always smaller. Comparing nums[mid] to nums[0] tells you which side of the rotation you're on.
