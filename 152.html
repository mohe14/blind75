Solution · Track Max AND Min Product
1
Init: maxProduct, minProduct, and result all start at nums[0].
2
Each step has 3 candidates for the new max and min — current element alone, extend max, or extend min:
tempMax = max(nums[i], maxP×nums[i], minP×nums[i])
minProduct = min(nums[i], maxP×nums[i], minP×nums[i])
result = max(result, tempMax)
i / nummaxPminPresult
0 / -2-2-2-2
1 / 33-63
2 / -424-1224 ✓
Why tempMax? Both max and min are derived from the same previous values. Computing minProduct first would overwrite values needed for maxProduct — tempMax saves the old max before it's clobbered.
