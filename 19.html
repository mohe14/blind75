Solution · Dummy Node + n+1 Gap
1
Dummy node: prepend temp = new ListNode(0) before head. Both first and second start at temp.
2
Advance first by n+1 steps (loop i=0; i<=n). The +1 ensures second lands on the predecessor of the node to delete — not the node itself.
gap after n+1 advance (n=2):
D → 1 → 2 → 3 → 4 → 5
↑second                 ↑first
3
Move both together until first == null. Fixed gap is preserved — second stops just before the target node.
4
Skip the target: second.next = second.next.next. Return temp.next — not head — to safely handle removing the head node.
Why dummy + n+1? Without a dummy, removing the head has no predecessor to update. The dummy handles it uniformly. Using n+1 (not n) ensures second is the predecessor, making the skip operation clean.
